整理的面试题目
一、 排序算法：

  1. 快速排序
    思路：挖坑填数+分治策略
    1．i =最左开始; j = 最右开始; 选择基准数的序号n，挖出来形成第一个坑a[n]。
    2． j–由后向前找比a[n]小的数，找到后挖出此数填到坑中，形成坑a[j]。
    3． i++由前向后找比它大的数，找到后也挖出此数填到坑中，形成坑a[i]。
    4．再重复执行2，3二步，直到i==j，将基准数填入a[j]中。
  2. 冒泡排序
    思路：相邻两数比较，大数向上冒泡
    优点：稳定 / 缺点：

  3. 直接插入排序
    思路：
    1. 对于一个数组A[0,n]的排序问题，假设认为数组在A[0,n-1]排序的问题已经解决了。
    2. 考虑A[n]的值，从右向左扫描有序数组A[0,n-1]，直到第一个小于等于A[n]的元素，将A[n]插在这个元素的后面
  4. 选择排序
    缺点：不稳定
    思路：
    1. 初始时在序列中找到最小（大）元素，放到序列的起始位置作为已排序序列；
    2. 再从剩余未排序元素中继续寻找最小（大）元素，放到已排序序列的末尾。
    3. 以此类推，直到所有元素均排序完毕。

  5. Shell排序
    思路：shell排序是把记录按下标的一定增量分组，对每组使用直接插入排序算法排序；随着增量逐渐减少，每组包含的关键词越来越多，当增量减至1时，整个文件恰被分成一组，算法便终止

二．查找算法：

  1. 二分查找
    思路：
    1. 对数列a[n]由小到大排序，最左的指针为Low，最右指针为High，High>Low，目标数据target；
    2. 判断a[(low+high)/2] == target 是否为成立，成立则a[(low+high)/2]即为目标数据；不成立，判断a[(low+high)/2]与target的大小；
    3. 如果a[(low+high)/2]>target，那么High指针指向序号(low+high)/2-1；如果a[(low+high)/2]<target，那么Low指针指向序号(low+high)/2+1；
    4. 重复2，3至找到目标数据。当Low指针>=High指针时，跳出循环，return None。
